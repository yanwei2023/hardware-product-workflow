# 硬件开发阶段模板

本系统必须通过模板自动生成项目阶段、必需工作包、交付物和阶段门条件，避免靠人手动创建任务导致遗漏。

模板文件：

```text
schemas/hardware-phase-template.json
```

## 模板原则

- 每个阶段必须有阶段门。
- 每个阶段门必须声明必需工作包。
- 每个必需工作包必须绑定 Agent 和人类角色。
- Agent 负责生成草稿、检查结果和风险提示。
- 人类角色负责审核和批准。
- 未批准的交付物不能用于阶段门通过。

## 第一版阶段

| 阶段 | 阶段门 | 重点 |
|---|---|---|
| 立项 | 立项评审 | 商业目标、初版计划、初版风险 |
| 需求冻结 | 需求冻结评审 | PRD、需求追踪、系统架构 |
| 设计冻结 | 设计冻结评审 | 硬件、结构、固件、BOM 风险 |
| EVT Exit | EVT Exit 阶段门 | EVT 测试计划、测试报告、问题关闭 |
| DVT Exit | DVT Exit 阶段门 | 可靠性、认证、DVT 测试 |
| PVT Exit | PVT Exit 阶段门 | 良率、工艺、供应准备 |
| 量产准备 | 量产放行评审 | 量产审核包、质量放行、最终摘要 |

## 生成项目时的动作

```text
选择流程模板
-> 创建项目
-> 创建阶段
-> 创建阶段门
-> 根据项目角色配对创建工作包
-> 为阶段门生成必需条件
-> 通知每个角色的 Agent 准备输入
```

## 阶段门检查逻辑

阶段门检查器不关心某个任务“看起来是否完成”，只关心正式证据：

- 是否存在必需工作包；
- 是否存在已批准交付物；
- 是否存在人类审核记录；
- 是否还有未处理高风险；
- 是否还有未处理高风险 Agent 发现项。

这保证了项目不会因为口头同步或手动打勾而绕过必要流程。
