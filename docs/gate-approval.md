# 阶段门批准与阶段推进

阶段门检查到 `READY` 之后，还不能自动进入下一阶段，必须由有权限的人类角色正式批准。

## 当前规则

批准阶段门需要满足：

- 阶段门检查结果为 `READY`；
- 操作者具有 `项目经理` 或 `阶段门批准人` 角色；
- 所有阶段门必需交付物已经被人类负责人批准；
- 高风险已经关闭或被有权限的人类接受。

## 阶段门审核包

阶段门页面和 API 会生成当前阶段门审核包，用于项目经理批准前检查：

- 必需工作包和交付物证据；
- 最新交付物状态；
- 对应人类审核人；
- 阶段门阻塞项；
- 当前阶段高风险及其是否阻塞阶段门；
- 是否已经具备批准条件。

## 状态变化

批准阶段门后：

```text
当前 Gate -> APPROVED
当前 Phase -> LOCKED
下一 Phase -> IN_PROGRESS / GATE_BLOCKED
项目 currentPhaseId -> 下一阶段
审计日志写入 GATE_APPROVED
```

如果没有下一阶段，项目状态进入 `COMPLETED`。

## 禁止行为

- Agent 不能批准阶段门；
- 阶段门未 READY 时不能批准；
- 无阶段门批准权限的人不能批准；
- 已批准的阶段门不能被普通阶段门检查覆盖状态。
