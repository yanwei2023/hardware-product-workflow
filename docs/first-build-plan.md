# 第一阶段搭建计划

## 第一步：流程主干

当前仓库已经包含：

- 核心架构说明；
- 流程主干说明；
- 共享 TypeScript 领域模型；
- 阶段门就绪检查器；
- 最小前端工作台骨架；
- 最小 Python Agent worker 协议；
- PostgreSQL 表结构草案；
- 本地基础设施配置。

当前已验证：

```text
node apps/api/src/demo.mjs
```

演示脚本会证明：在 EVT 测试计划、EVT 报告、EVT 问题关闭计划未被人类批准，或高风险未被处理前，EVT Exit 阶段门会保持阻塞；当所有必需交付物被人类批准，并且高风险被接受或关闭后，阶段门变为可通过。

当前也已验证：Agent 输出缺少模板必填章节时，会被标记为 `OUTPUT_INVALID`，工作包退回 `NEEDS_AGENT_REVISION`，不会进入人类审核。

## 第二步：真实 API

下一步实现目标：

- 创建项目接口；
- 创建阶段模板接口；
- 根据模板生成必需工作包；
- 请求 Agent 执行接口；
- 提交 Agent 输出接口；
- 创建人类审核接口；
- 运行阶段门就绪检查接口。

## 第三步：第一条端到端演示链路

演示场景：

```text
创建智能控制器项目
-> 生成 EVT 阶段工作包
-> Test Agent 提交 EVT 报告草稿
-> 测试工程师批准测试计划，但尚未批准 EVT 报告
-> 阶段门检查器阻塞 EVT Exit
-> 高热风险被人类接受或关闭
-> EVT 报告被批准
-> 阶段门变为可通过
-> 项目经理批准 EVT Exit
```
