# 最小权限模型

当前试用版先使用本地演示用户，不接登录系统。

用户配置文件：

```text
schemas/demo-users.json
```

## 当前规则

### 工作包审核

用户可以参与审核的条件：

- 是工作包绑定的人类负责人；或
- 具备交付物模板注册表中声明的 `requiredReviewRoles` 之一。

### 工作包批准

批准比普通审核更严格。

只有工作包绑定的人类负责人可以执行：

```text
APPROVE
APPROVE_WITH_CONDITIONS
```

原因：批准后的交付物会进入阶段门证据链，必须由该角色负责人承担责任。

### 风险接受与关闭

当前允许以下角色接受或关闭风险：

- 项目经理
- 质量负责人
- 阶段门批准人

其他角色可以提出风险处理建议，但不能接受或关闭风险。

## 后续增强

后续接入真实用户系统后，需要支持：

- 登录；
- 部门与组织；
- 项目成员；
- 多级审批；
- 阶段门批准委员会；
- 临时授权；
- 权限审计。
